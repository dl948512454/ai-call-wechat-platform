# AI外呼加微协同平台 MVP PRD

## 1. 产品定位

AI外呼加微协同平台是CRM之外的AI外呼协同能力层，不负责线索导入、线索清洗、客户主数据维护和销售全流程管理。

CRM主平台负责线索手动导入、自动导入、客户主数据和最终客户资产沉淀。本平台只负责AI外呼规则、AI外呼过程状态、线索流转人工规则、人工承接结果和关键状态回流。

## 2. MVP目标

MVP验证一条主链路：

CRM线索进入AI外呼范围 -> 命中自动AI外呼规则 -> AI完成外呼和意向识别 -> 命中转人工规则 -> 人工完成外呼/加微/反馈 -> 按规则回流CRM。

## 3. 平台边界

### 3.1 CRM负责

- 线索手动导入和自动导入。
- 客户主数据维护。
- 线索归属销售/BPO同学。
- 客户基础状态。
- 最终客户资产沉淀。

### 3.2 本平台负责

- 自动AI外呼规则配置。
- 线索是否进入AI外呼的规则判断。
- AI外呼执行状态展示。
- AI结果结构化沉淀。
- 线索流转人工规则配置。
- AI转人工队列。
- 人工承接动作和加微反馈。
- CRM回流规则配置和状态回流。

## 4. 用户角色

### 4.1 运营

配置自动AI外呼规则、转人工规则和CRM回流规则，查看AI外呼任务执行情况。

### 4.2 BPO主管

查看AI转人工队列，分配高意向客户，处理超时接管。

### 4.3 销售/BPO同学

承接AI筛选出的客户，完成电话确认、短信加微、RPA加微和结果反馈。

## 5. MVP功能

### 5.1 工作台

展示AI外呼协同链路核心指标和待处理任务。

核心指标：

- 今日AI外呼量。
- AI接通率。
- 有效交互率。
- 高意向率。
- 同意加微率。
- 人工接管数。
- 加微成功数。

筛选能力：

- 业务线，目前仅新车加微。
- AI话术。
- 日期范围。

工作台列表：

- AI外呼任务执行概览。
- 待人工接管任务。
- 规则命中和转人工概览。

### 5.2 规则配置

#### 5.2.1 AI外呼规则配置

用于配置“哪些同学/团队的哪些CRM线索”自动进入AI外呼。

配置项：

- 规则名称。
- 适用同学/团队。
- CRM线索条件，例如已分配、未加微、未联系。
- AI话术。
- 转人工规则。
- 启用/停用状态。

说明：

- 外呼时间和复呼逻辑由专门平台配置，本平台不重复建设。
- 手动外呼任务由CRM主平台导入或自动下发，本平台不提供手动导入入口。

典型规则：

- BPO一组名下“已分配且未加微、未联系”的线索，自动发起AI外呼。
- 若AI外呼完成后，客户高意向但未加微，自动转人工高优先级接管外呼。

#### 5.2.2 转人工规则配置

用于配置AI外呼后哪些线索流转给人工承接。

规则条件：

- 客户通话标签，例如高意向、价格敏感、反感、需要资料、愿意加微。
- 加微状态，例如未加微、同意加微、加微失败。
- 历史沟通记录，例如近7天未人工跟进、历史多次未接通、历史拒绝。
- 客户等级，例如高价值、普通、风险客户。

规则结果：

- 是否转人工。
- 任务等级。
- 承接团队。
- SLA时效。
- 推荐动作。

#### 5.2.3 回流CRM规则配置

用于配置哪些状态需要回流CRM。

回流字段：

- AI外呼状态。
- 是否转人工。
- 人工承接状态。
- 加微结果。
- 失败原因。

### 5.3 线索管理

展示客户在AI外呼与人机协同链路中的状态。

展示字段：

- 客户ID。
- 联系状态。
- 加微状态。
- AI意向等级。
- 跟进人。
- 清洗员。
- 线索创建信息。
- 最近AI外呼结果。
- 转人工状态。

详情能力：

- 查看AI外呼记录。
- 查看人工操作记录。
- 查看历史跟进记录。
- 查看CRM回流状态。

### 5.4 AI转人工队列

AI外呼后，满足规则的客户进入人工承接队列。

转人工条件：

- 客户同意加微。
- AI识别为高意向。
- AI识别为中意向且规则要求人工确认。
- 高价值客户需要人工复核。

筛选项：

- 跟进人。
- 清洗员。
- 任务时间。
- 客户ID。
- 任务等级。

队列展示：

- 客户姓名、客户ID、脱敏手机号。
- 来源任务。
- AI意向等级。
- SLA剩余时间。
- 当前承接人。
- 通话记录和历史跟进记录。
- AI摘要。
- 推荐动作。
- 任务等级。
- 任务时间。

人工可执行动作：

- 人工外呼。
- 短信加微。
- RPA加微。
- 反馈结果。
