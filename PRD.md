# 外呼加微协同平台 MVP PRD

## 1. 产品定位

AI外呼加微协同平台是CRM之外的AI外呼协同能力层，不负责线索导入、线索清洗、客户主数据维护和销售全流程管理。

CRM主平台负责线索手动导入、自动导入、客户主数据和最终客户资产沉淀。本平台只负责AI外呼规则、AI外呼过程状态、线索流转人工规则、人工承接结果和关键状态回流。

## 2. MVP目标

MVP验证一条主链路：

CRM线索进入AI外呼范围 -> 命中自动AI外呼规则 -> AI完成外呼和意向识别 -> 命中转人工规则 -> 人工完成外呼/加微/反馈 -> 按规则回流CRM。

## 3. 平台边界

### 3.1 CRM负责

- 线索手动导入和自动导入。
- 客户主数据维护。
- 线索归属销售/BPO同学。
- 客户基础状态。
- 最终客户资产沉淀。

### 3.2 本平台负责

- 自动AI外呼规则配置。
- 线索是否进入AI外呼的规则判断。
- AI外呼执行状态展示。
- AI结果结构化沉淀。
- 线索流转人工规则配置。
- 人员配置，使用电话作为唯一ID管理BPO和销售同学。
- 线索详情内人工接管。
- 人工承接动作和加微反馈。
- CRM回流规则配置和状态回流。

## 4. 用户角色

### 4.1 运营

配置自动AI外呼规则、转人工规则和CRM回流规则，查看AI外呼任务执行情况。

### 4.2 BPO主管

查看线索任务池中的待接管线索，在详情页完成高意向客户接管和超时处理。

### 4.3 销售/BPO同学

承接AI筛选出的客户，完成电话确认、短信加微、RPA加微和结果反馈。

## 5. MVP功能

### 5.1 总览

展示AI外呼协同链路核心指标和待处理任务。

核心指标：

- 今日外呼量。
- 外呼完成量。
- 接通率。
- 通话时长。
- 有效交互率。
- 高意向率。
- 同意加微率。
- 人工接管数。
- 加微成功数。

全局筛选能力：

- 左侧业务方向选择，支持新车加微、二手车加微。
- 业务方向切换后影响总览、线索任务池、人员配置和规则配置的上下文。
- 总览支持按外呼方式、日期范围、跟进人筛选。

总览列表：

- 外呼加微漏斗数据。
- 待处理展示线索任务池中的P0高优待跟进线索数和P1择机跟进线索数。

### 5.2 规则配置

#### 5.2.1 AI外呼规则配置

用于配置“哪些同学/团队的哪些CRM线索”自动进入AI外呼。

配置项：

- 规则名称。
- 适用同学/团队。
- CRM线索条件，例如已分配、未加微、未联系。
- 触发方式，支持自动触发和定时触发。
- 定时触发规则，例如每天固定时间、每2小时、每周固定时间。
- AI话术。
- 转人工规则。
- 启用/停用状态。

说明：

- 外呼时间和复呼逻辑由专门平台配置，本平台不重复建设。
- 线索任务由CRM主平台导入或自动下发，本平台不提供手动导入入口。

典型规则：

- BPO一组名下“已分配且未加微、未联系”的线索，自动发起AI外呼。
- 若AI外呼完成后，客户高意向但未加微，自动转人工高优先级接管外呼。

#### 5.2.2 转人工规则配置

用于配置AI外呼后哪些线索流转给人工承接。

规则条件：

- 客户通话标签，例如高意向、价格敏感、反感、需要资料、愿意加微。
- 加微状态，例如未加微、同意加微、加微失败。
- 历史沟通记录，例如近7天未人工跟进、历史多次未接通、历史拒绝。
- 客户等级，例如高价值、普通、风险客户。

规则结果：

- 是否转人工。
- 任务等级。
- 承接团队。
- SLA时效。
- 推荐动作。

#### 5.2.3 回流CRM规则配置

用于配置哪些状态需要回流CRM。

回流字段：

- AI外呼状态。
- 是否转人工。
- 人工承接状态。
- 加微结果。
- 失败原因。

### 5.3 线索任务池

展示客户在AI外呼与人机协同链路中的状态。

展示字段：

- 客户ID。
- 联系状态。
- 外呼次数。
- 加微状态。
- AI意向等级。
- 跟进人。
- 清洗员。
- 线索创建信息。
- 最近AI外呼结果。
- 转人工状态。

筛选能力：

- 联系状态。
- 加微状态。
- 转人工优先级：AI外呼中-无需接管、P0高优待跟进、P1择机跟进、已跟进。
- 大模型通话标签，例如客户反感、是否已买车、AI通话问题。
- 跟进人。

详情能力：

- 点击线索可查看线索详情。
- 查看客户买车意图信息，包括意向车型、购车城市、购车预算、购车周期、购车类型。
- 查看每通AI电话后的大模型标签，包括客户反感、是否已买车、AI通话问题、语音助理、加微方式、最后节点、有效交互等。
- 查看历史外呼记录，并区分AI外呼和人工外呼。
- 多次外呼需要展示多次通话结果和摘要。
- 查看人工操作记录。
- 查看历史跟进记录。
- 查看CRM回流状态。
- 线索详情以页面右侧抽屉形式展示。
- 在详情抽屉中进行人工接管操作。

### 5.4 线索详情人工接管

AI外呼后，满足规则的客户在详情页展示人工接管区，由人工完成后续动作。

转人工条件：

- 客户同意加微。
- AI识别为高意向。
- AI识别为中意向且规则要求人工确认。
- 高价值客户需要人工复核。

接管区展示：

- 任务等级。
- 当前承接人。
- 推荐动作。
- 接管状态。

人工可执行动作：

- 转人工。
- 人工外呼。
- 短信加微。
- RPA加微。
- 跟进反馈。

跟进反馈字段：

- 客户状态。
- 是否同意加微。
- 下次跟进时间。
- 是否回流CRM。
- 备注。

### 5.5 人员配置

用于管理BPO和销售同学，并通过电话作为唯一ID关联人员、线索和人工承接任务。

配置字段：

- 电话，作为人员唯一ID。
- 人员姓名。
- 角色，例如BPO、销售、主管。
- 团队分组。
- 性别。
- 启用/停用状态。

说明：

- 线索任务池中的跟进人、承接人需要通过电话唯一识别。
- 当CRM下发线索归属人时，平台优先使用电话匹配人员配置。
- 电话重复时不允许新增人员。
